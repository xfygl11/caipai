# 个人助手 APP 重构方案 v3

> 基于 Build74 (个人助手_v9.9_Build74.zip) 重构为标准 Android Gradle 项目
> 完整交互路径：每个可点击位置的页面跳转和内容展示

---

## 一、核心原则

1. **彩票模块零修改**：nc-lottery.js 全部 1266 行代码原样保留，不做任何改动
2. **所有 Web Assets 零修改**：main.html、app.js、nc-movie-engine.js、nc-repo.js 等全部文件原样复制到 assets/www/
3. **内置 9 个采集源**：从 nc-repo.js 提取默认采集源列表，在原生侧预置到 localStorage
4. **本地数据库**：创建 Room 数据库，采集数据按采集源+分类保存到 Room + IndexedDB
5. **自动采集**：内置源和后期添加的采集源在添加时就自动采集分类数据

**选集列表结构**（`openMovieModal()` in nc-movie-engine.js:827）：
- 按 `vod_play_from` 分组（如 "线路1$$$线路2"）
- 每组内按 `#` 分隔的集数（如 "第1集$http://..."#"第2集$http://..."）

### 5.2 自动采集流程

**触发时机**：采集源添加时（内置源初始化 + 用户自定义添加）

**流程**：
```
1. 保存 source 到 Room 数据库
   ↓
2. HTTP GET {base}?ac=detail
   → 获取分类列表 + 推荐影片
   ↓
3. 保存分类到 categories 表
   → 去重：type_pid=0 的为根分类
   ↓
4. 保存推荐影片到 movies 表
   → source_id + category='推荐'
   ↓
5. 对每个根分类，HTTP GET {base}?ac=detail&t={type_id}&pg=1
   → 获取该分类下的影片列表
   ↓
6. 批量保存到 movies 表
   → source_id + category={分类名} + vod_id
   → 自动去重：vod_id 已存在则跳过
   ↓
7. 更新 source.last_sync_time
   ↓
8. 通知 WebView 更新 UI（通过 JS Bridge）
```

**采集策略**：
- 每个分类最多采集 100 部新影片
- 每页 20 条，自动翻页直到取满 100 部或无更多数据
- 内置源在首次启动时自动执行完整采集
- 自定义源在添加时自动执行完整采集
- 用户可手动触发重新采集（"我的"页面 → 打开采集窗口）

### 5.3 WebView IndexedDB 与 Room 的关系

| 层级 | 存储内容 | 用途 |
|------|---------|------|
| **Room（原生）** | 采集源、分类、影片数据 | 持久化存储、自动采集、数据管理 |
| **IndexedDB（WebView）** | 收藏、历史、搜索历史、彩票数据 | Web 端功能运行 |
| **SharedPreferences** | 配置源 URL、当前选中源 | 快速读取配置 |

2. [ ] 启动后自动采集 9 个内置源，Room 数据库有数据
3. [ ] 主页 Tab：影视分类导航、内容网格、搜索、播放全部正常
4. [ ] 彩票 Tab：8 种彩票 + 三角洲计算器全部正常（nc-lottery.js 零修改）
5. [ ] 历史/收藏 Tab：收藏和历史功能正常
6. [ ] 我的 Tab：采集窗口、数据库查看、配置管理正常
7. [ ] 长按主页 Tab：打开仓库管理面板，可切换/添加/删除采集源
8. [ ] 站点选择面板：点击任意源可切换
9. [ ] 更多菜单：配置源输入、扫码、预设源、导入导出均可用
10. [ ] 视频播放弹窗：artplayer + HLS 正常播放，集数切换、倍速、全屏可用
11. [ ] 数据采集弹窗：选择源后开始采集，进度和日志正常显示
12. [ ] 数据库查看器：按源和分类筛选，分页浏览正常
13. [ ] 种子数据：网络不可用时自动降级到 ffzy_seed.js
14. [ ] 无崩溃、无明显卡顿
15. [ ] nc-lottery.js 代码未做任何修改（diff 为 0）

## 十、Git 提交流程

```
feat: 个人助手重构 - 项目脚手架
  - 创建 Android Gradle 项目
  - 复制全部 Web Assets（零修改）
  - 配置 WebViewAssetLoader

feat: 数据库与采集源系统
  - Room 数据库（sources/categories/movies）
  - 预置 9 个内置采集源
  - AutoCollector 自动采集
  - AndroidJSBridge 桥接

feat: 彩票模块验证
  - nc-lottery.js 零修改运行
  - 8 种彩票 + 计算器验证
```

## 十一、技术栈

| 项目 | 技术 |
|------|------|
